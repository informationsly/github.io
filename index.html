##正则
如果你使用正则表达式处理的字符串中有大于两个字节长度(0xFFFF)的字符，正则表达式匹配模式必须加u这个标记位。
正则表达式中的.并不能匹配所有的字符，只能匹配长度小于两个字节的字符，若要匹配大于两个字节的字符，需要在u修饰符下才可以。
换行符，回车符，行分割符，段分割符，需要使用s修饰符，ES6还没有实现。

y修饰符和g修饰符的区别

y修饰符和g修饰符都是全局匹配的，后一次匹配都从上一次匹配成功的下一个位置开始。不同之处在于，g修饰符只要剩余位置中存在匹配就可，而y修饰符确保匹配必须从剩余的第一个位置开始。

##数据类型扩展
Set的用法， Map的用法， WeakSet的用法， WeakMap的用法

Map和Object的区别：Object的key只能是字符串，而Map的key可以是任意数据类型（数组，引用对象都可以作为key）

WeakSet和Set的区别：支持的数据类型不一样，WeakSet只支持对象（字符串也不行），WeakSet存储的只是一个值的引用，它也不会去检测这个值是否被垃圾回收了，WeakSet没有clear方法，不能遍历，没有size属性(WeakMap也一样)。

##Proxy和Reflect
proxy可以对目标对象起到保护作用，不被外部访问到，Object的方法移植到了Reflect上。调用Object的方法都可以使用Reflect代替。

安装hexo

安装hexo之前要求电脑上要安装了Node.js和Git，可以分别使用node –version和git –version检查一下电脑上是否安装了这两个程序，安装Node.js、Git、hexo参照hexo文档就可以了，还是蛮详细的。

使博客能够通过域名访问

博客搭建好后，我希望它能够从互联网访问到，具体做法如下：
1.在GitHub上新建一个github.io的代码仓库
2.修改_config.yml文件中的配置
deploy:
type: git
repo: git@github.com:<你自己的github用户名>/github.io.git
3.安装git插件hexo-deployer-git

4.部署到github

5.配置自己的域名
如何与一个域名做关联，可以参看这个GitHubHelp,大致的过程就是我们要有一个域名，并做一个域名解析，把域名指向192.30.252.153，192.30.252.154，并在代码仓库下创建一个名字为CNAME的文件，内容就是你的域名名称，文章中说到的A record就是一个记录类型，指将域名指向一个IPv4地址。
域名可以在万网上购买，也不是太贵，我买的时候是29元/年，购买成功后你就可以在万网后台做域名解析了。设置了域名解析后，过一段时间，就可以在浏览器里通过域名访问博客了。
备注：CNAME文件我们需要把它下载下来放在$ hexo init 命令所指定文件夹的source目录下，这样你下次博客内容改变重新部署后CNAME文件不会丢失。
