WeakSet和Set的区别：支持的数据类型不一样，WeakSet只支持对象（字符串也不行），WeakSet存储的只是一个值的引用，它也不会去检测这个值是否被垃圾回收了，WeakSet没有clear方法，不能遍历，没有size属性(WeakMap也一样)。

响应式网站设计

响应式网站设计的概念

响应式网站是一个设计概念，它是多项技术的综合体

由来

2010年Ethan Marcotte在Responsive Web Design一文中提出了响应式网站设计的概念

用到的技术

  • flexible grid layout 弹性(流式)网格布局
  • flexible image 弹性图片
  • media queries 媒体查询（css2.1出现的概念，css3被强化）

优点

  • 每个设备都能得到正确的设计
  • 搜索优化
  • 更好的用户体验

缺点

  • 会加载更多的样式和脚本资源
  • 设计比较难精确定位和控制
  • 老版本浏览器兼容不好

视口

布局视口

布局视口为网页PC端版式，为默认布局模式下的页面显示大小

可视视口

可视视口为手机端能看见PC端网页的显示大小（默认是设备的宽度）：如同放大镜离书本距离变化后，放大镜里能看到的内容的多少，不能简单理解为设备屏幕的大小。

理想视口

理想视口就是为构建手机浏览器优化的页面而添加的。
理想视口（就是布局视口在一个设备上的最佳尺寸）应该理解为一个状态值，

如何使用理想视口：
在页面中添加如下元标签(视口元标签)：

user-scalable=no，禁止用户缩放

布局视口的宽度和设备可视视口的宽度一样时，这个布局视口就是一个理想视口，此时布局视口和可视视口达到一致。

网页中的单位

  • px
    1个px相当于1个像素

  • em
    相对的长度单位(em依赖与父元素的字体大小)
    1.em 相对参照物为父元素的font-size
    2.em 具有继承的特点
    3.当继承链上所有的父元素都没有设置font-size时，浏览器会有一个默认的em设置：1em = 16px
    em的缺点：各个em元素的父元素的字体大小不确定，容易造成混乱

  • rem
    1.rem的相对参照物为根元素html,相对参照固定不变，所以比较好计算
    2.当没有html没有设置font-size时，浏览器会有一个默认的rem设置：1rem = 16px，这点与em是一致的
    3.IE8及以下的IE并不支持rem属性

当font-size设置为62.5%(10/16*100%)时，1rem = 10px, 便于计算
font-size:62.5% 1rem = 10px
font-size:100% 1rem = 16px

使博客能够通过域名访问

博客搭建好后，我希望它能够从互联网访问到，具体做法如下：
1.在GitHub上新建一个github.io的代码仓库
2.修改_config.yml文件中的配置
deploy:
type: git
repo: git@github.com:<你自己的github用户名>/github.io.git
3.安装git插件hexo-deployer-git

4.部署到github

5.配置自己的域名
如何与一个域名做关联，可以参看这个GitHubHelp,大致的过程就是我们要有一个域名，并做一个域名解析，把域名指向192.30.252.153，192.30.252.154，并在代码仓库下创建一个名字为CNAME的文件，内容就是你的域名名称，文章中说到的A record就是一个记录类型，指将域名指向一个IPv4地址。
域名可以在万网上购买，也不是太贵，我买的时候是29元/年，购买成功后你就可以在万网后台做域名解析了。设置了域名解析后，过一段时间，就可以在浏览器里通过域名访问博客了。
备注：CNAME文件我们需要把它下载下来放在$ hexo init 命令所指定文件夹的source目录下，这样你下次博客内容改变重新部署后CNAME文件不会丢失。
